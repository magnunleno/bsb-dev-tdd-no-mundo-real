TDD - Test Driven Development

  • Não vou me aprofundar nos conceitos;
  • Kent Beck popularizou o conceito no XP em 1999;
  • Processo de desenvolvimento de software;
  • Lifecycle simplificado
    1. Transforme as especificações em casos de tests;
    2. Implemente somente a funcionalidade para validar o caso de testes;
    3. Refatore (se necessário)
    4. GOTO 1

Etapas

  • Planeje
    • Novas funcionalidades...
    • Modificações de funcionalidades existentes...
    • API (não é a API REST)...
  • Escreva os Testes
    • Unitários e Funcionais
  • Escreva o Código
    • Até que todos os testes passem!
  • REFATORE!

Testando... 123...

  • Não faça testes manuais;
  • Utilize dados fictícios;
  • ...ou extraídos de produção (descaracterizados);
  • Faça mocking;
  • Valide toda sua lógica (cobertura);
  • Se está difícil de testar, você não está fazendo certo!
